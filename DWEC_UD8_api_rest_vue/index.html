    API RESTful

    Concepte API REST

    API REST

    Una API REST (Representational State Transfer) és un conjunt d'arquitectures de comunicació que es defineixen per a la creació, lectura, actualització i eliminació (CRUD) d'informació. Es basa en els principis d'arquitectura RESTful, que són un conjunt de convencions i restriccions que faciliten la creació de sistemes web escalables, mantenibles i interoperables.

    A continuació, es detallen alguns dels principals conceptes associats a les API REST:

    • Recursos: En una API REST, les dades o les entitats es representen com a recursos, que poden ser qualsevol cosa (com ara un usuari, una publicació, una imatge, etc.). Cada recurs té una URI (Identificador de Recurs Uniforme) únic que s'utilitza per accedir-hi.

    • Operacions CRUD: Les operacions CRUD (Crear, Llegir, Actualitzar, Esborrar) es mapegen a les operacions HTTP: POST per crear, GET per llegir, PUT o PATCH per actualitzar i DELETE per esborrar. Aquestes operacions es realitzen sobre els recursos identificats per les URIs.

    • Representació: Les representacions dels recursos es poden transferir entre el client i el servidor en diferents formats com ara JSON o XML. La representació defineix l'estat actual del recurs.

    • Estil arquitectònic: REST és un estil arquitectònic que destaca la simplicitat, la modularitat, l'escalabilitat i la visibilitat. Utilitza les restriccions com la falta d'estat (stateless), la interfície uniforme, la capa de client-servidor, etc.

    • Interfície uniforme: Les API REST segueixen una interfície uniforme que inclou operacions comuns, com ara la identificació del recurs, la manipulació a través de representacions i l'enllaç entre recursos.

    Les API REST són àmpliament utilitzades a l'arquitectura de serveis web i són preferides per molts desenvolupadors per la seva simplicitat i eficàcia. El protocol HTTP és la base de moltes implementacions d'API REST, però REST no està lligat a cap protocol específic.

    Els end-points

    Pas 1: Disseny de l'API

    Per començar, defineix quins recursos vols gestionar amb la teva API. En aquest exemple, utilitzarem una aplicació de gestió de tasques i definirem el recurs "tasca".

    Pas 2: Defineix els Recursos

    Decideix quines propietats tindrà cada tasca. En aquest exemple, podem tenir les propietats "id", "nom", "descripció" i "completada".

    Pas 3: Defineix les Operacions CRUD

    Associa cada operació CRUD amb un mètode HTTP. Per a la nostra aplicació de tasques, podríem tenir:

    • GET /tasques: Obtenir totes les tasques.
    • GET /tasques/:id: Obtenir una tasca per la seva ID.
    • POST /tasques: Crear una nova tasca.
    • PUT /tasques/:id: Actualitzar una tasca per la seva ID.
    • DELETE /tasques/:id: Eliminar una tasca per la seva ID.

    Pas 4: Implementa els Punts Finals

    // Importa les dependencies
const express = require('express');
const app = express();
app.use(express.json());

// Dades d'exemple
let tasques = [
  { id: 1, nom: 'Comprar al supermercat', descripcio: 'Llet, pa, fruita', completada: false },
  { id: 2, nom: 'Estudiar per l'examen', descripcio: 'Repasar apunts', completada: false },
];

// Punt final per obtenir totes les tasques
app.get('/tasques', (req, res) => {
  res.json(tasques);
});

// Punt final per obtenir una tasca per la seva ID
app.get('/tasques/:id', (req, res) => {
  const tascaId = parseInt(req.params.id);
  const tasca = tasques.find(t => t.id === tascaId);

  if (!tasca) {
    res.status(404).json({ missatge: 'Tasca no trobada' });
  } else {
    res.json(tasca);
  }
});

// Punt final per crear una nova tasca
app.post('/tasques', (req, res) => {
  const novaTasca = req.body;
  tasques.push(novaTasca);
  res.status(201).json(novaTasca);
});

// Punt final per actualitzar una tasca per la seva ID
app.put('/tasques/:id', (req, res) => {
  const tascaId = parseInt(req.params.id);
  const tascaIndex = tasques.findIndex(t => t.id === tascaId);

  if (tascaIndex === -1) {
    res.status(404).json({ missatge: 'Tasca no trobada' });
  } else {
    tasques[tascaIndex] = req.body;
    res.json(tasques[tascaIndex]);
  }
});

// Punt final per eliminar una tasca per la seva ID
app.delete('/tasques/:id', (req, res) => {
  const tascaId = parseInt(req.params.id);
  tasques = tasques.filter(t => t.id !== tascaId);
  res.json({ missatge: 'Tasca eliminada correctament' });
});

// Inicia el servidor
const port = 3000;
app.listen(port, () => {
  console.log(`Servidor en funcionament a http://localhost:${port}`);
});

    Aquest codi utilitza Express per gestionar les crides i simula l'emmagatzematge de tasques en memòria. Les operacions CRUD es gestionen a través dels diferents punts finals.

    Pas 5: Prova la teva API

    Prova els teus punts finals utilitzant eines com Postman, curl o fins i tot creant un frontend interactiu.

    • Obtenir totes les tasques: GET http://localhost:3000/tasques
    • Obtenir una tasca per la seva ID: GET http://localhost:3000/tasques/1
    • Crear una nova tasca: POST http://localhost:3000/tasques (amb dades al cos de la petició)
    • Actualitzar una tasca per la seva ID: PUT http://localhost:3000/tasques/1 (amb dades actualitzades al cos de la petició)
    • Eliminar una tasca per la seva ID: DELETE http://localhost:3000/tasques/1
