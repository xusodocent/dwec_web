    UD2 Javascript Events

    1. Que són els events?

    En el model de programació que heu vist fins ara, les instruccions s'executaven una darrere l'altra fins a acabar el programa. Les aplicacions web actuals requerixen interacció amb l'usuari i amb accions que ocorren a la pàgina, com per exemple prèmer una tecla, passar el ratolí sobre una secció o esperar a que es carregue la pàgina per a interactuar amb el DOM.

    Aquestes accions són el que anomenem events i són de gran utilitat per als programadors ja que doten a l'aplicació de interacció amb l'usuari.

    Que són els events?

    Els events són notificacions que envia la pàgina per a notificar que alguna cosa ha ocorregut. Els events estan representats per objectes de la interfaz Event.

    ;

    2. Tipus d'events

    Els events estan nomenats amb el prefixe on de manera que l'event que correspondria a fer click amb el ratolí seria onclick, polsar una tecla onkeypress, seleccionar un element onfocus, etc. Cada element podrà proporcionar diferents tipus d'events. Podem consultar una llista més completa d'events a la web de la 3wschool.

    3. Manejadors d'events

    Els events són controlats per porcions de codi que s'encarreguen de gestionar que ocorre a la pàgina en cas que l'event es produisca. Aquest control es pot dur a terme de vàries formes.

    Com a atributs dintre del HTML

    JavaScript
    1
    <button onclick="alert('Hello!')">Saludar</button>
    Com es veu es crea un atribut amb el nom de l'event i el codi a executar. No és recomanable utilitzar aquest mètode ja que embruta el codi HTML i és tediós gestionar events amb codi de més d'una línia.

    Amb la variable this

    La variable this permet reduir el codi ja que ens proporciona una referencia a l'element propi. Vegem dos porcions de codi que realitzen el mateix.

    Sense la variable this.

    Amb Listeners: addEventListener() i removeEventListener()

    És un mètode que permet assignar una funció per a manejar una funció a un event determinat. A diferència de les anteriors, es pot assignar en temps d'execució de la pàgina. Imaginem que creem contingut dinàmicament en la nostra web, estil SPA. Volem assignar unes funcions per a controlar uns elements select però aquestos elements no existeixen en carregar-se la pàgina, sinò que els creem dinàmicament degut a alguna acció de l'usuari (exemple típic de dos selects anidats de província i poble). Com que no existeix el select en el moment de crear la pàgina no podem assignar una funció que controle l'event onchange que provoca el select.

    Vegem com es resol amb els Listeners.

    JavaScript
    1
2
3
4
5

    function assignarEvents(){
    selects=document.getElementsByTagName('select');
    for(j=0;j<selects.length;j++){
        selects[j].addEventListener("change", revisa_select);
    }
}

    D'aquesta forma hem assignat el control de l'event onchange de tots els select de la pàgina a una funció anomenada revisa_select.

    function resalta(elEvento) {
  var evento = elEvento || window.event;
  switch(evento.type) {
    case 'mouseover':
      this.style.borderColor = 'black';
      break;
    case 'mouseout':
      this.style.borderColor = 'silver';
      break;
  }
}

window.onload = function() {

  document.getElementById("seccion").onmouseout = resalta;

    Capturing

    Funciona a la inversa que l'anterior. Anirà recorrent l'element més genèric fins el més específic.

    Podem triar com volem que es comporte el nostra manejador d'events. Si el handler és true es capturarà l'event el el capturing, si és false es capturarà en el bubbling.

    JavaScript
    1
2
3
    elem.addEventListener(..., {capture: true})
// or, just "true" is an alias to {capture: true}
elem.addEventListener(..., true)
