    Mòduls en Javascript

    Mòduls

    Els mòduls en JavaScript són una forma d'organitzar i reutilitzar el codi en diferents arxius. Són especialment útils en aplicacions grans o complexes on vols separar el codi en parts més petites i gestionables.

    Per utilitzar mòduls en JavaScript, segueix aquesta guia:

    Creació d'un Mòdul:

    • Defineix el teu mòdul en un fitxer JavaScript separat. Per exemple, pots tenir un fitxer anomenat mimodul.js.
    mimodul.js
    1
2
3
4
5
6
    // mimodul.js
export function funcioDelModul() {
    return "Això és una funció del mòdul.";
}

export const unaVariable = 42;

    En aquest exemple, hem exportat una funció i una variable del mòdul.

    Importació d'un Mòdul:

    • Ara, pots importar aquest mòdul a un altre fitxer JavaScript on vulguis fer servir les funcions o variables definides al mòdul. Per fer-ho, utilitza la paraula clau import.
    unaltrefitxer.js
    1
2
3
4
5
    // unaltrefitxer.js
import { funcioDelModul, unaVariable } from './mimodul.js';

console.log(funcioDelModul()); // Això és una funció del mòdul.
console.log(unaVariable); // 42

    Assegura't que l'adreça del fitxer del mòdul sigui correcta, i utilitza l'extensió .js si cal. En aquest exemple, hem importat la funció funcioDelModul i la variable unaVariable des del mòdul mimodul.js.

    Ús del Mòdul:

    • Ara pots fer servir les funcions i les variables importades dins del fitxer que has fet la importació, tal com es mostra a l'exemple anterior. Això et permet utilitzar funcions i dades del mòdul en altres parts de la teva aplicació.

    Els mòduls són una característica estàndard a partir d'ECMAScript 6 (ES6) i són compatibles amb molts navegadors moderns i entorns de desenvolupament. Això et permet organitzar i estructurar el teu codi de manera més eficaç i fàcil de mantenir.

    Local Storage

    El Local Storage és una característica de l'emmagatzematge web en JavaScript que permet emmagatzemar dades en el navegador web d'un usuari de manera persistent. Aquesta emmagatzematge és similar a les cookies, però amb algunes diferències claus:

    • Capacitat: El Local Storage té una capacitat més gran que les cookies, permetent emmagatzemar més dades (generalment fins a 5-10 MB per domini, encara que això pot variar entre navegadors).

    • Persistència: Les dades emmagatzemades al Local Storage romanen en el navegador després de tancar-lo i reiniciar-lo. Les cookies poden tenir una data d'expiració.

    • Seguretat: Les dades del Local Storage no s'envien al servidor en cada sol·licitud HTTP, cosa que succeeix amb les cookies. Això pot ser més segur per a les dades de l'usuari, ja que no es comparteixen constantment amb el servidor.

    Per emmagatzemar dades al Local Storage, pots utilitzar les següents funcions en JavaScript:

    • localStorage.setItem(key, value): Aquesta funció permet emmagatzemar un valor associat a una clau. La clau és una cadena i el valor pot ser qualsevol dada JavaScript vàlida (generalment una cadena).

    • localStorage.getItem(key): Aquesta funció permet recuperar el valor emmagatzemat associat a una clau específica.

    • localStorage.removeItem(key): Aquesta funció elimina una entrada emmagatzemada a partir de la seva clau.

    Per exemple:

    JavaScript
    1
2
3
4
5
6
7
8
9
    // Emmagatzemar un valor
localStorage.setItem('nom', 'John');

// Recuperar un valor
var nom = localStorage.getItem('nom');
console.log(nom); // John

// Eliminar un valor
localStorage.removeItem('nom');

    El Local Storage és útil per emmagatzemar dades com configuracions de l'usuari, preferències, estadístiques i altres dades que vulguis mantenir localment en el navegador. No obstant això, has de ser conscient que les dades del Local Storage són accessibles des de JavaScript en la mateixa pàgina web, pel que no és el lloc adequat per emmagatzemar dades sensibles o privades.

    Cookies

    Les cookies

    Les cookies són petites peces de dades que es poden emmagatzemar en el navegador d'un usuari i que s'envien al servidor amb cada sol·licitud HTTP. Aquestes són àmpliament utilitzades en desenvolupament web per emmagatzemar informació i estat en el costat del client. Les cookies es poden utilitzar en JavaScript mitjançant l'objecte document.cookie.

    Aquí tens una descripció bàsica de com funcionen les cookies:

    Emmagatzemar una Cookie:

    Per emmagatzemar una cookie des de JavaScript, has d'assignar un valor a document.cookie. El valor ha de ser una cadena en el format "nom=valor". Pots afegir tantes cookies com vulguis, separant-les per punts i coma.

    Exemple:

    JavaScript
    1
2
    document.cookie = "usuari=John";
document.cookie = "preferencia=color=blau";

    Llegir Cookies:

    Per llegir les cookies existents, pots accedir a document.cookie i obtenir la cadena de totes les cookies com una llista separada per punts i coma. Després, pots dividir aquesta cadena per aconseguir els valors individuals.

    Exemple:

    JavaScript
    1
2
3
4
5
6
7
    var totesLesCookies = document.cookie;
var cookiesSeparades = totesLesCookies.split('; ');
for (var i = 0; i < cookiesSeparades.length; i++) {
    var cookie = cookiesSeparades[i];
    var [nom, valor] = cookie.split('=');
    console.log(nom + ": " + valor);
}

    Modificar Cookies:

    Per modificar una cookie existent, simplement reassigna el seu valor mitjançant document.cookie. És important tenir en compte que si vols modificar una cookie, has de proporcionar totes les seves propietats (nom, valor, data d'expiració, etc.) de nou, ja que aquesta assignació reemplaçarà completament la cookie existent.

    Exemple:

    JavaScript
    1
    document.cookie = "usuari=NouValor; expires=Thu, 01 Jan 2099 00:00:00 UTC";

    Eliminar Cookies:

    Per eliminar una cookie, assigna-li una data d'expiració que sigui anterior a la data i hora actual. Això fa que la cookie sigui invàlida i s'eliminarà.

    Exemple:

    JavaScript
    1
    document.cookie = "usuari=; expires=Thu, 01 Jan 1970 00:00:00 UTC";

    Restriccions de Cookies:

    Les cookies estan subjectes a diverses restriccions de seguretat i privadesa. Les cookies només poden ser llegides pel mateix domini que les va crear i estan limitades en capacitat. També has de tenir en compte les preferències de privadesa de l'usuari.

    En resum, les cookies són una manera de persistir dades al navegador de l'usuari i enviar-les de tornada al servidor amb cada sol·licitud HTTP. Malgrat ser ampliament utilitzades, és important tenir en compte les restriccions de seguretat i privadesa quan les utilitzis. Moltes vegades, es fan servir altres mètodes més segurs i eficaços per a emmagatzemar dades al costat del client, com el Local Storage o les Web Storage API.

    Geolocalització

    La API de geolocalització en JavaScript proporciona una forma d'accedir a la ubicació del dispositiu des del navegador. Això permet a les aplicacions web obtenir la ubicació de l'usuari, cosa que és útil per a una varietat de casos d'ús, com serveis de mapes, recomanacions basades en la ubicació i molt més. Aquí et explico com funciona:

    Sol·licitud de permisos:

    Abans d'accedir a la ubicació de l'usuari, has de sol·licitar el seu permís. Els navegadors moderns demanen permís a l'usuari abans de proporcionar la ubicació. Pots fer-ho utilitzant la funció navigator.geolocation.getCurrentPosition o navigator.geolocation.watchPosition.

    Obtenció de la ubicació actual:

    Per obtenir la ubicació actual, pots utilitzar navigator.geolocation.getCurrentPosition. Aquí tens un exemple:

    JavaScript
    1
2
3
4
5
    navigator.geolocation.getCurrentPosition(function(position) {
    const latitud = position.coords.latitude;
    const longitud = position.coords.longitude;
    console.log(`Latitud: ${latitud}, Longitud: ${longitud}`);
});

    Aquesta funció pren una funció de devolució de trucada que s'executarà un cop s'obtingui la ubicació. Les dades de la ubicació es passen com a arguments a la funció de devolució de trucada.

    Actualització de ubicació contínua:

    Pots utilitzar navigator.geolocation.watchPosition si necessites seguir la ubicació de l'usuari en temps real. Aquesta funció també pren una funció de devolució de trucada i s'executarà cada cop que s'actualitzi la ubicació de l'usuari.

    JavaScript
    1
2
3
4
5
    const watchId = navigator.geolocation.watchPosition(function(position) {
    const latitud = position.coords.latitude;
    const longitud = position.coords.longitude;
    console.log(`Latitud: ${latitud}, Longitud: ${longitud}`);
});

    Pots aturar el seguiment de la ubicació utilitzant navigator.geolocation.clearWatch(watchId).

    Gestió d'errors:

    Tingues en compte que l'obtenció de la ubicació de l'usuari pot no estar sempre disponible, i és important gestionar els possibles errors. La funció de devolució de trucada també pot rebre un objecte PositionError que descriu qualsevol error que ocorri.

    JavaScript

    navigator.geolocation.getCurrentPosition(function(position) {
    // Èxit
}, function(error) {
    switch (error.code) {
        case error.PERMISSION_DENIED:
            console.error("L'usuari ha denegat el permís de ubicació.");
            break;
        case error.POSITION_UNAVAILABLE:
            console.error("La ubicació no està disponible.");
            break;
        case error.TIMEOUT:
            console.error("Temps d'espera exhaurit en intentar obtenir la ubicació.");
            break;
        case error.UNKNOWN_ERROR:
            console.error("S'ha produït un error desconegut.");
            break;
    }
});

    Requisits de seguretat:

    És important destacar que, per utilitzar la API de geolocalització, el teu lloc web ha d'oferir-se a través de HTTPS en la majoria dels navegadors moderns. A més, tingues en compte la privadesa de l'usuari i sol·licita permís abans d'accedir a la seva ubicació.

    La API de geolocalització en JavaScript proporciona una forma potent d'accedir a la ubicació de l'usuari, però també requereix responsabilitat i consideració de la privadesa de l'usuari.

    API de Mapbox

    Mapbox és una plataforma de mapes que ofereix una API rica per a la visualització de mapes personalitzats en aplicacions web i mòbils. Per utilitzar la API de Mapbox, has de seguir aquestes passes bàsiques:

    Crear un compte de Mapbox:

    Abans de començar, hauràs de registrar-te a Mapbox i obtenir un access token, que serà necessari per autenticar-te quan facis sol·licituds a la seva API. Pots obtenir un access token a través del lloc web de Mapbox.

    Afegir el Mapbox SDK al teu projecte:

    Si estàs desenvolupant una aplicació web, hauràs d'afegir el SDK de Mapbox al teu projecte. Mapbox ofereix biblioteques JavaScript per a aquest propòsit. Pots incloure aquestes biblioteques al teu projecte utilitzant un gestor de paquets com npm o afegint una etiqueta de script directament al teu HTML.

    Exemple d'inclusió de la biblioteca JavaScript de Mapbox a través de CDN:

    HTML
    1
2
    <script src='https://api.mapbox.com/mapbox-gl-js/v2.6.1/mapbox-gl.js'></script>
<link href='https://api.mapbox.com/mapbox-gl-js/v2.6.1/mapbox-gl.css' rel='stylesheet' />

    Crear un contenidor d'aplicació:

    Mapbox utilitza un contenidor HTML per a la visualització del mapa. Hauràs d'afegir un element HTML, com un div, al teu document HTML que servirà com a contenidor del mapa.

    Exemple d'un div de contenidor:

    HTML
    1
    <div id='map' style='width: 100%; height: 400px;'></div>

    Inicialitzar el mapa amb JavaScript:

    Amb el SDK de Mapbox inclòs al teu projecte, pots inicialitzar i configurar el mapa amb JavaScript. Has d'utilitzar el teu access token per autenticar-te.

    Exemple de com inicialitzar el mapa:

    JavaScript
    1
2
3
4

6

8
    mapboxgl.accessToken = 'YOUR_ACCESS_TOKEN';

var map = new mapboxgl.Map({
    container: 'map', // ID de l'element HTML que actua com a contenidor
    style: 'mapbox://styles/mapbox/streets-v11', // Estil del mapa
    center: [-74.5, 40], // Coordenades centrals
    zoom: 9 // Nivell de zoom
});

    Personalitzar el mapa:

    Pots personalitzar el mapa de Mapbox afegint capes, punts d'interès, rutes i moltes altres característiques utilitzant les funcions proporcionades pel SDK de Mapbox.

    Afegir interacció i funcionalitat:

    Pots afegir interacció a través d'events, com ara clics, moviments del ratolí i molts altres. També pots afegir funcionalitats com ara geocodificació i direccions mitjançant les API de Mapbox.

    Mapbox ofereix una ampla documentació i recursos en línia per a ajudar-te a aprofundir en la seva API i crear mapes personalitzats per a les teves aplicacions. També ofereix l'ús de Mapbox Studio per a dissenyar estils de mapes personalitzats.

    Template literals

    En JavaScript, els "template literals" (o plantilles literals) són una característica introduïda amb ECMAScript 6 (ES6) que permeten crear cadenes de text de manera més concisa i amb funcionalitats addicionals. Utilitzen la sintaxi de cometes invertides (``) en lloc de les cometes simples o dobles.

    Aquí tens un exemple bàsic d'ús de template literals:

    JavaScript
    1
2
3
4
5
6
7
    const nom = "John";
const edat = 30;

// Utilització de template literals
const missatge = `Hola, el meu nom és ${nom} i tinc ${edat} anys.`;

console.log(missatge);

    En aquest exemple, la cadena de text està envoltada per cometes invertides, i les variables es poden integrar directament dins la cadena utilitzant ${variable}. Això fa que sigui més llegible i convenient que la concatenació tradicional de cadenes o l'ús de l'operador +.

    Les avantatges principals dels template literals inclouen:

    1. Interpolació de variables: Les variables es poden afegir fàcilment a la cadena sense necessitat d'operadors de concatenació.

    2. Suport per expressions: Pots incrustar expressions dins les plantilles literals, no només variables. Aquestes expressions són avaluades i incrustades a la cadena.

    JavaScript
    1
2
3
4
5
    const a = 5;
const b = 10;

const resultat = `La suma de ${a} i ${b} és ${a + b}.`;
console.log(resultat); // La suma de 5 i 10 és 15.
    1. Multilínia: Pots crear cadenes multilínia sense necessitat de concatenar múltiples cadenes o utilitzar caràcters d'escapament.
    JavaScript
    1
2
3
4
5
    const multilinea = `
Línia 1
Línia 2
Línia 3
`;

    En resum, els template literals són una forma més elegant i versàtil de treballar amb cadenes de text a JavaScript, especialment quan es necessita incorporar variables o expressions dins les cadenes.

    Aplicar canvis en arrays

    Funció map()

    La funció map en JavaScript és un mètode que es pot utilitzar en arrays per iterar sobre cada element i aplicar una funció proporcionada a cada element de l'array original, creant així un nou array amb els resultats de les crides a aquesta funció.

    La signatura de la funció map és la següent:

    JavaScript
    1
    array.map(callback(currentValue, index, array), thisArg);
    • callback: Funció que es crida per cada element de l'array, amb tres arguments:
    • currentValue: Valor de l'element actual de l'array.
    • index: Índex de l'element actual.
    • array: L'array que està sent iterat.
    • thisArg (opcional): Valor per a utilitzar com a "this" al executar la funció de retorn.

    Aquí tens un exemple senzill de com funciona la funció map:

    JavaScript
    1
2
3
4
5
6
7
8
    const nombres = [1, 2, 3, 4, 5];

// Utilitzant map per duplicar cada element de l'array
const duplicats = nombres.map(function(num) {
  return num * 2;
});

console.log(duplicats); // Resultat: [2, 4, 6, 8, 10]

    En aquest exemple, la funció passada com a argument a map duplica cada element de l'array original. El resultat es guarda en un nou array (duplicats), sense modificar l'array original (nombres).

    map és útil quan vols transformar cada element d'un array sense modificar l'array original. També facilita la lectura i el manteniment del codi en comparació amb l'ús de bucles for.

    Drag and drop

        const draggableElement = document.getElementById('draggableElement');
        const dropzone = document.getElementById('dropzone');

        // Event listener per iniciar l'arrossega
        draggableElement.addEventListener('dragstart', function (event) {
            // Guardar l'identificador de l'element arrossegat
            event.dataTransfer.setData('text/plain', draggableElement.id);
        });

        // Event listener per permetre el soltar
        dropzone.addEventListener('dragover', function (event) {

        dropzone.addEventListener('drop', function (event) {
            event.preventDefault(); // Prevenir el comportament per defecte que impedeix el soltar

            const draggedElementId = event.dataTransfer.getData('text/plain');

            // Obtindre l'element arrossegat
            const draggedElement = document.getElementById(draggedElementId);

            // Afegir l'element arrossegat a la dropzone
            dropzone.appendChild(draggedElement);
        });
    </script>

</body>

</html>

    Objectes

    Creació

    En JavaScript, es poden crear objectes de diverses maneres. Aquí tens algunes de les maneres més comunes de crear objectes:

    1. Object Literal (Literals d'objecte): És la forma més simple i comuna de crear un objecte. Utilitza la notació d'objecte literal, que consisteix en col·locar parells de claus i valors entre les claus.

    Herència

    En JavaScript, l'herència es realitza mitjançant els prototips. Cada objecte té un prototip, i els objectes poden heretar propietats i mètodes del seu prototip. A continuació, et mostraré com funciona l'herència en JavaScript utilitzant els prototips i les classes.

    // Prototip base
function Animal(nom) {
  this.nom = nom;
}

// Afegim un mètode al prototip d'Animal
Animal.prototype.saluda = function() {
  console.log(`Hola, sóc un animal anomenat ${this.nom}.`);
};

// Funció constructora que hereta de Animal
function Gos(nom, raça) {
  // Cridem al constructor de l'Animal amb la paraula clau "call"
  Animal.call(this, nom);
  this.raça = raça;
}

// Establim la relació de prototip entre Gos i Animal
Gos.prototype = Object.create(Animal.prototype);
Gos.prototype.constructor = Gos;  // Corregim el constructor ja que s'ha sobrescrit amb Object.create

// Afegim un mètode addicional al prototip de Gos
Gos.prototype.lladra = function() {
  console.log('Guau, guau!');
};

// Creem una instància de Gos
const firulais = new Gos('Firulais', 'Golden Retriever');

// Fem servir els mètodes heretats
firulais.saluda(); // Hola, sóc un animal anomenat Firulais.
firulais.lladra();  // Guau, guau!

    En aquest exemple, Gos hereta de Animal mitjançant Object.create(Animal.prototype). Això significa que els objectes creats amb la funció constructora Gos tindran un prototip que apunta al prototip d'Animal.

    // Classe base
class Animal {
  constructor(nom) {
    this.nom = nom;
  }

  saluda() {
    console.log(`Hola, sóc un animal anomenat ${this.nom}.`);
  }
}

// Classe que hereta d'Animal
class Gos extends Animal {
  constructor(nom, raça) {
    super(nom); // Cridem al constructor de l'Animal amb la paraula clau "super"
    this.raça = raça;
  }

  lladra() {
    console.log('Guau, guau!');
  }
}

// Creem una instància de Gos
const firulais = new Gos('Firulais', 'Golden Retriever');

// Fem servir els mètodes heretats
firulais.saluda(); // Hola, sóc un animal anomenat Firulais.
firulais.lladra();  // Guau, guau!

    En aquest exemple, Gos hereta de Animal amb la paraula clau extends. La paraula clau super s'utilitza per cridar el constructor de la classe pare (Animal) dins del constructor de la classe filla (Gos). Això garanteix que l'herència es gestioni correctament.
