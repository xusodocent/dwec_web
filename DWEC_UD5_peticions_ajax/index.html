    UD5 AJAX Peticions

    En aquest document es mostren les diferents formes existents per a fer peticions AJAX a un servidor. Totes les formes són vàlides i el programador escollirà la forma en que li resulta més còmode treballar.

    1. XMLHttpRequest()

    L'objecte XMLHttpRequest és una interfície que s'utilitza en programació web per fer peticions a servidors web mitjançant JavaScript. Aquesta interfície permet als desenvolupadors web comunicar-se amb servidors remots sense necessitat de recarregar la pàgina web, la qual cosa fa possible crear aplicacions web més dinàmiques i interactives.

    Amb l'objecte XMLHttpRequest, pots fer peticions HTTP, com ara GET o POST, i rebre dades de resposta del servidor. Això és útil per carregar contingut dinàmic, com arxius JSON o HTML, o per enviar dades a un servidor per ser processades. Aquesta funcionalitat és àmpliament utilitzada en aplicacions web AJAX (Asynchronous JavaScript and XML), que permeten actualitzar parts específiques d'una pàgina web sense necessitat de recarregar-la completament.

    Actualitat

    Tot i que l'objecte XMLHttpRequest va ser molt important en el passat, avui en dia molts desenvolupadors web utilitzen l'API Fetch, que proporciona una interfície més moderna i potser més fàcil d'utilitzar per fer peticions HTTP. L'objecte XMLHttpRequest, tot i això, encara és vàlid i pot ser útil en certs casos, sobretot en projectes més antics o en situacions on és necessària una compatibilitat amb navegadors més antics.

    2. FETCH

    L'API Fetch en JavaScript no és específicament "en AJAX" (Asynchronous JavaScript and XML), sinó que és una API moderna per fer peticions de xarxa asíncrones. L'ús de Fetch ha reemplaçat en gran mesura l'ús de l'objecte XMLHttpRequest per fer peticions HTTP a servidors remots en aplicacions web. Amb Fetch, pots realitzar peticions a servidors web i obtenir respostes de manera asíncrona, el que permet carregar o enviar dades sense necessitat de recarregar tota la pàgina web.

    L'API Fetch ofereix una interfície més simple i prometedora que XMLHttpRequest. Proporciona una forma més moderna d'interactuar amb recursos a través d'HTTP i altres protocols de xarxa. Aquí tens un exemple senzill de com fer una sol·licitud GET amb Fetch:

    JavaScript
    1
2
3
4
    fetch('https://exemple.com/api/dades')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

    En aquest exemple, fem una sol·licitud GET a una URL especificada i, quan es rep la resposta, la convertim a JSON i la mostrem a la consola. La interfície Promise utilitzada aquí facilita la gestió de peticions asíncrones i respostes.

    L'API Fetch és compatible amb les últimes versions de navegadors web moderns i és una opció recomanada per fer peticions de xarxa en aplicacions web actuals. Proporciona una sintaxi més clara i més flexible que l'objecte XMLHttpRequest, i permet fer servir Promises per gestionar les respostes, cosa que fa que el codi sigui més llegible i mantenible.

    3 AXIOS

    Axios és una llibreria JavaScript que s'utilitza per fer peticions HTTP des del costat del client en aplicacions web. Aquesta llibreria és particularment popular en el desenvolupament web, ja que simplifica la realització de peticions a servidors web i la gestió de les respostes obtingudes mitjançant una interfície fàcil d'utilitzar.

    Axios està basat en Promises i ofereix una forma més simple i concisa de fer peticions HTTP que l'objecte XMLHttpRequest o l'API Fetch. La llibreria gestiona molts dels aspectes complicats de les peticions de xarxa, com la gestió de les respostes, l'enviament d'encapçalaments i la gestió d'errors. També permet fer peticions amb diferents mètodes HTTP com GET, POST, PUT, DELETE, etc.

    Aquí tens un exemple bàsic de com fer una sol·licitud GET amb Axios:

    JavaScript

  .then(response => {

  })
  .catch(error => {
    console.error('Error:', error);
  });

    Axios també és molt útil en aplicacions de JavaScript del costat del client com React, Vue.js i Angular, ja que facilita la integració amb servidors i l'obtenció de dades a través d'APIs.

    En resum, Axios és una llibreria popular i àmpliament utilitzada per fer peticions HTTP a servidors web des del costat del client en desenvolupament web, i proporciona una interfície més senzilla i fàcil d'utilitzar per aquest propòsit.

    5. ASYNC / AWAYT

    Es pot utilitzar asincronia d'aquesta forma...

    JavaScript
    1
2
3
4
5
6
7
    const cargarDatos = async () => {
  const url = "https://jsonplaceholder.typicode.com/todos/1";
  const res = await fetch(url);
  const datos = await res.json();
  console.log(datos);
};
cargarDatos();

    async i await són característiques de JavaScript que es van introduir amb l'arribada de les Promises i ECMAScript 2017 (ES8). Aquestes característiques són utilitzades per simplificar la gestió de codi asíncron en JavaScript, fent-lo més llegible i fàcil d'entendre. Són especialment útils quan es fan peticions de xarxa o quan es realitzen operacions que triguen un temps indeterminat a completar-se, com ara la lectura de fitxers o l'accés a bases de dades.

    • async: és una paraula clau que s'afegeix davant d'una funció per indicar que aquesta funció conté codi asíncron. Les funcions marcades com a async sempre retornen una Promise i permeten l'ús de await dins del cos de la funció.

    Exemple d'una funció async:

    JavaScript
    1
2
3
    async function ferAlgunaFeina() {
  // Codi asíncron aquí
}
    • await: és una paraula clau que s'utilitza dins de funcions marcades com a async per indicar que es vol esperar fins que una Promise es resolgui (o es rebutgi) abans de continuar amb la següent línia de codi. Això fa que el flux d'execució es detingui fins que la Promise estigui en un estat final (ja sigui resolta o rebutjada).

    Exemple d'ús de await:

    JavaScript
    1
2
3
4
    async function exemple() {
  const resultat = await promesaAsincrona(); // Espera fins que la promesa es resolgui
  console.log(resultat);
}

    Note

    L'ús de async i await facilita la gestió de Promises i fa que el codi sigui més semblant a una programació síncrona, el que el fa més llegible i menys propens a "callback hell" o "Pyramid of Doom", que són problemes comuns quan es treballa amb codi asíncron anidat. Aquestes característiques han millorat en gran mesura la forma de gestionar la programació asíncrona a JavaScript.
