    Cicle de vida

    El cicle de vida d'un component en Vue 3 fa referència a les diferents etapes o fases per les quals passa un component des de la seva creació fins a la seva destrucció. Aquestes etapes permeten als desenvolupadors executar codi en moments específics del cicle de vida del component. Aquí tens una descripció general de les principals etapes del cicle de vida d'un component en Vue 3:

    Creació (Creation):

    • beforeCreate: Aquest esdeveniment es dispara abans que les instàncies del component siguin inicialitzades.
    • created: Es dispara després que s'ha creat l'instància del component. Les dades, mètodes, observadors i esdeveniments han estat inicialitzats i vinculats.

    Actualització (Updating):

    • beforeMount: Es dispara abans que el component sigui muntat (mounted) al DOM. En aquest punt, el template ja s'ha compilat a una funció de renderització.
    • mounted: Es dispara quan el component s'ha muntat al DOM. És un bon lloc per a fer operacions que necessitin accedir als elements del DOM.
    • beforeUpdate: Es dispara abans que les dades reatives del component es reprocessin i el DOM es torni a renderitzar.
    • updated: Es dispara quan les dades reatives es reprocessen i el DOM s'actualitza.

    Destrucció (Destroying):

    • beforeUnmount: Es dispara abans que el component sigui desmuntat del DOM.
    • unmounted: Es dispara quan el component s'ha desmuntat del DOM. És un bon moment per alliberar recursos i realitzar tasques de neteja.

    Errors (Error Handling):

    • errorCaptured: Es dispara quan es produeix un error en qualsevol dels fills d'aquest component. Es pot utilitzar per gestionar i gestionar errors a nivell de component.

    Composició (Composition API):

    • setup: Aquest esdeveniment permet utilitzar la nova Composició API de Vue 3. S'executa abans de totes les altres fases i és on es configuren els mètodes, dades i altres elements del component.

    Aquestes etapes permeten als desenvolupadors controlar el comportament del component en diferents moments del seu cicle de vida. Per exemple, es podria utilitzar l'esdeveniment created per inicialitzar dades o carregar recursos, o l'esdeveniment beforeUnmount per alliberar recursos abans que el component sigui desmuntat.

    <template>
  <div>
    <h2>Dades de l'API</h2>
    <ul>
      <li v-for="item in dadesAPI" :key="item.id">{{ item.nom }}</li>
    </ul>
  </div>
</template>

<script>
export default {
  data() {
    return {
      dadesAPI: [],
    };
  },
  mounted() {
    // L'esdeveniment mounted es dispara quan el component s'ha muntat al DOM
    // Fem una crida a una API o realitzem altres operacions aquí
    this.obtenirDadesAPI();
  },
  methods: {
    obtenirDadesAPI() {
      // Simulem una crida a una API per obtenir dades
      // En una aplicació real, això seria una crida real a una API
      setTimeout(() => {
        this.dadesAPI = [
          { id: 1, nom: 'Element 1' },
          { id: 2, nom: 'Element 2' },
          { id: 3, nom: 'Element 3' },
        ];
      }, 1000); // Simulem un retard de 1 segon
    },
  },
};
</script>

    En aquest exemple:

    • Quan el component es munta al DOM, l'esdeveniment mounted s'activa automàticament.
    • Dins de la funció mounted, cridem el mètode obtenirDadesAPI que simula una crida a una API per obtenir dades.
    • Les dades obtingudes de la crida a l'API s'assignen a la propietat dadesAPI, i aquesta propietat s'utilitza a la plantilla per mostrar els elements de la llista.

    Aquest patró és comú quan vols realitzar accions específiques, com ara crides a API, manipulació del DOM o altres operacions, després que el component s'ha muntat al DOM.
